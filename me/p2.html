Trombone

Haskell REST PostgreSQL Middleware

The Trombone data access server streamlines adaptation of relational SQL schemas to mobile-friendly JSON-APIs operating within the RESTful web service paradigm. Or, in plain Swedish:

Trombone är en robust serverapplikation som möjliggör smidig anpassning av SQL-databaser till REST/JSON-gränssnitt, lämpliga för mobila webblösningar och hybrida applikationer där större delen av programlogiken ligger på klienten.

How it works

Rules are layed out by a set of route templates:

# Create a new customer
#   e.g. POST /customer { "name"     : "ACME", 
#                         "phone"    : "555-BIZNESS", 
#                         "industry" : "Robotics" }
POST /customer  <>
  INSERT INTO customers (name, phone, industry)
    VALUES ({{name}}, {{phone}}, {{industry}})

The route format is given by the following, simplified grammar.

<route> ::= <method> <uri> <symbol> <action>

The server instantiates the template during dispatch, replacing placeholder variables with actual data, supplied in the request body and uri components.

Trombone is written in Haskell and available for use under the BSD license.

QtZeroConf

C++ Qt ZeroConf

QtZeroConf is a high-level Zeroconf interface for C++/Qt-based client-server applications, leveraging the Observer pattern and Qt's signals/slots mechanism. It is layered over Avahi — a technology which implements the Apple Zeroconf specification to facilitate service discovery on a local network via the mDNS/DNS-SD protocol suite.

Ett Zeroconf-gränssnitt för klient-serverprogramvara baserad på ramverket Qt. Zeroconf förenklar uppgiften att hitta lokala nätverkstjänster såsom skrivare och fildelningsprogram över ett LAN. Programbiblioteket använder sig av Qt:s signal/slots-mekanism för att söka asynkront efter tillgängliga tjänster med hjälp av s.k. multicast DNS och DNS Service Discovery. Lösningen bygger på Avahi, som är en fri implementation av Apples Zeroconf-specifikation.
